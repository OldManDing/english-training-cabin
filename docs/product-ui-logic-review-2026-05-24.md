# 英语训练舱产品与 UI 逻辑审查清单

日期：2026-05-24

## 闭环主线

目标闭环应为：

```text
目标设置 -> 入门诊断 -> 今日任务 -> 练习作答 -> 评分反馈 -> 错因入队 -> 主动复习 -> 能力画像更新 -> 下一次任务调度
```

## 本次发现与处理

| 优先级 | 问题 | 影响 | 处理状态 |
| --- | --- | --- | --- |
| P0 | 今日训练页弱项诊断卡片使用静态示例数据，没有反映真实诊断或练习结果 | 用户无法判断系统推荐是否来自真实学习证据 | 已修复：今日页读取 `SkillProfile` 并展示真实分数/待诊断状态 |
| P0 | 入门诊断没有设置每日可投入时间，完成后固定写入 45 分钟 | 目标设置与任务调度不一致，计划解释不完整 | 已修复：诊断流程可选择 30/45/60/90 分钟，并写入 `StudyGoal.dailyMinutes` |
| P0 | 入门诊断完成后 `prioritySkills` 固定排序 | 今日任务不一定优先安排真实弱项 | 已修复：按诊断分数从低到高生成训练优先级 |
| P0 | 复习页中优先级项目点击后只弹 toast，没有进入复习流程 | 复习队列不能形成完整可执行闭环 | 已修复：真实复习项的中优先级操作可进入主动复盘流程 |
| P1 | 今日时间卡片始终显示“高效模式” | 策略切换反馈不一致 | 已修复：随策略切换显示“高效模式/巩固模式” |
| P1 | 今日突破进展卡片显示静态“5 -> 2” | 容易误导用户，以为已有真实训练进展 | 已修复：改为展示真实训练证据增长 |
| P1 | 写作/翻译依赖 AI 评阅 | 供应商异常可能中断主观题训练 | 已验证：服务端 `/api/ai/evaluate-subjective` 已有离线兜底 |
| P1 | 题库来源边界需要明确 | 上线后存在版权误解风险 | 已覆盖：产品内有版权声明，材料导入经过后端校验 |

## 追加复审与修正

| 优先级 | 问题 | 影响 | 处理状态 |
| --- | --- | --- | --- |
| P0 | 今日页在只有部分能力证据时，对未训练模块显示默认分数 | 用户会误以为系统已经诊断过听力或翻译 | 已修复：每个能力点必须有真实 `SkillProfile` 才显示分数，否则显示“待诊断” |
| P0 | 复习页无真实错因时仍展示体验样例 | 生产环境容易把演示数据误认为个人错因 | 已修复：无复习项时改为空状态，引导先完成训练生成错因 |
| P0 | 口语“开始第二次重说”会直接生成报告 | 口语重说流程缺少真实二次产出步骤 | 已修复：新增第二次重说转写确认，提交后才写入能力画像和复习队列 |
| P1 | 入门诊断结果页存在未验证话术，如平台排名、词汇量和固定权重 | 影响产品可信度 | 已修复：改为基于用户自评分数动态生成强项、弱项和推荐依据 |
| P1 | 部分本地规则诊断被标成 AI 分析 | 混淆本地算法和外部 AI 供应商能力 | 已修复：阅读、听力、计划调度等本地规则改为“错因/智能任务调度”表述 |
| P1 | 练习能力画像重复训练时覆盖证据数 | 趋势和证据增长不稳定 | 已修复：练习完成写入 `SkillProfile` 时按证据数合并分数和 confidence |

## 当前功能逻辑判断

| 功能 | 逻辑状态 | 说明 |
| --- | --- | --- |
| 入门诊断 | 闭环可用 | 首次无能力证据时今日任务优先推荐诊断，诊断写入目标、每日时间和五项能力画像 |
| 今日任务 | 闭环可用 | 根据目标、复习队列、能力画像和策略生成任务，可直接跳转诊断、复习、听力、阅读、写作、翻译、口语 |
| 阅读练习 | 闭环可用 | 选择题作答、信心标记、反馈解析、生成 attempt/reviewItem/skillProfile |
| 听力练习 | 闭环可用 | 播放/原文/作答/信心/错因反馈，完成后写入训练证据 |
| 写作/翻译 | 闭环可用 | 作答后请求 AI 评阅，服务端失败时返回离线结构化反馈，完成后写入能力画像和复习项 |
| 口语重说 | 闭环可用 | 录音或文本转写、AI/离线反馈、二次重说转写确认、写入口语能力和复习队列 |
| 复习队列 | 闭环可用 | 真实错因项按步骤主动复盘，完成后更新掌握度、下次复习时间和复习能力证据；无真实错因时显示空状态 |
| 能力地图 | 闭环可用 | 从本地能力画像读取分数，暂无证据时明确提示先诊断或训练 |
| 题库/材料 | MVP 可用 | 内置原创样题、AI 生成模拟题、JSON 导入和后端结构校验；不是完整授权题库系统 |

## 仍不建议扩大宣传的边界

- 当前是单用户、本地优先产品，不是账号云同步产品。
- 当前题库是原创样题、AI 模拟题和用户自导入材料，不是官方真题库。
- 当前诊断是轻量自评 + 能力画像初始化，不是正式标准化测评。
- CET-6/IELTS/TOEFL 只保留架构扩展边界，当前上线不应承诺完整可用。

## 上线验收口径

- 产品口径：可作为“英语训练舱 / AI 英语能力训练系统”的 MVP 上线试用，首发聚焦 CET-4，但不要宣传为完整题库或官方真题系统。
- UI 口径：核心页面可达、关键 CTA 能执行、空状态和隐私/版权/AI 兜底有说明。
- 测试口径：上线前必须通过 `npm.cmd run lint`、`npm.cmd run test`、`npm.cmd run build`、`npx.cmd playwright test`。
