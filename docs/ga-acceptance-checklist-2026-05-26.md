# 英语训练舱 GA 验收清单

日期：2026-05-26

## 1. 验收原则

GA 验收不能只看页面是否能打开，也不能只看接口烟测是否返回 200。必须按真实用户旅程验证：

- 用户不看开发说明，也能知道第一步该做入门诊断。
- 用户完成诊断后，系统必须产生真实能力证据，而不是保存自评分。
- 今日任务必须根据目标、能力画像、复习队列和词汇证据生成。
- 每个训练入口必须能作答、评分、写入记录、生成复习项或能力画像。
- 移动端 320px、平板 768px、桌面宽屏都不能出现按钮遮挡或横向溢出。

## 2. 用户旅程验收

| 旅程 | 必须通过的行为 | 阻塞标准 |
| --- | --- | --- |
| 首次进入 | 今日训练显示“入门能力诊断”，无证据时最高优先级是诊断 | 首屏像营销页，或不知道下一步 |
| 入门诊断 | 能返回上一步；包含阅读、听力播报、翻译、写作、口语初筛；不同答案产生不同分数 | 只有自评、不能返回、没有写入证据 |
| 今日任务 | 诊断后根据最低技能、到期复习、词汇证据生成任务 | 静态推荐，或任务按钮不能进入真实模块 |
| 阅读/听力 | 能作答、标记信心、查看错因、写入 attempts 和 skillProfiles | 只展示材料，不保存学习记录 |
| 写作/翻译 | 用户先产出答案，再得到结构化反馈和复习项 | 直接给答案，或反馈不能落库 |
| 口语重说 | 完成首次表达、反馈、二次重说、对比报告 | 只有录音按钮，没有二次闭环 |
| 词汇练习 | 能播放单词和例句，完成词义辨析，低信心或错误项进复习队列 | 没有词汇入口或没有语音播报 |
| 复习队列 | 主动回忆、挖空补全、语境化输出、更新下次复习时间 | 只是查看解析 |
| 能力地图 | 无证据显示待诊断；有证据才显示分数和预测 | 展示伪造分数 |

## 3. 数据证据验收

每个闭环完成后至少检查 IndexedDB 这 4 类记录：

- `practiceSessions`：本次训练 session。
- `attempts`：每题或每次输出的作答记录。
- `reviewItems`：错题、低信心、薄弱表达或词汇复习项。
- `skillProfiles`：能力画像证据和 evidenceCount。

入门诊断必须额外检查：

- `studyGoals.examDate` 默认新用户为 `2026-06-13`。
- `studyGoals.prioritySkills` 包含 `vocabulary`。
- 诊断报告产生 5 条 attempts 和 5 条 skillProfiles。

## 4. UI 验收

- “AI 英语能力教练”不得作为悬浮遮挡按钮；入口文案统一为“入门诊断与能力画像”或“入门能力诊断”。
- 专项练习顶部必须展示基础材料规模。
- 词汇练习必须是一等入口，不只能藏在复习或设置里。
- 所有主按钮触控高度不低于 44px。
- 320px 手机宽度下不允许横向滚动；导航可以横向滚动，但页面主体不能溢出。
- 弹窗、Toast、顶部操作区不得遮挡当前页面的关键 CTA。

## 5. 测试门禁

上线前必须通过：

```bash
npm.cmd run lint
npm.cmd run test
npm.cmd run build
npx.cmd playwright test tests/e2e/mvp-smoke.spec.ts tests/e2e/mobile-smoke.spec.ts
```

生产发布后必须再跑：

```bash
$env:SMOKE_BASE_URL='https://study.xmlga.top'; npm.cmd run smoke:production
$env:SMOKE_BASE_URL='https://study.xmlga.top'; npm.cmd run smoke:ga
```

如果其中任意一项失败，不能声明 GA。
