# 词句记忆方法落地记录

日期：2026-05-25

## 结论

之前产品已经具备“诊断 -> 练习 -> 错因入队 -> 间隔复习”的基础闭环，但词汇和语句记忆还不够强：复习页更像流程确认，没有强制用户主动回忆、挖空提取和输出使用。

本次已把核心学习方法落到数据和 UI：

- 主动回忆：复习项包含 `recallPrompt` 和 `recallAnswer`，复习时必须先输入回忆内容。
- 语块化：从阅读定位句、听力解析、口语升级句、写作/翻译参考表达中提取 `chunks`。
- 挖空补全：复习项生成 `clozePrompt` 和 `clozeAnswer`。
- 语境化输出：复习完成前必须造句、复述或翻译。
- 间隔重复：复习间隔升级为 1/3/7/14/30 天梯度，并记录 `retrievalCount`。

## 产品闭环

```text
练习作答
-> 错因/低信心/AI反馈
-> 生成 MemoryReviewTask
-> 主动回忆
-> 挖空补全
-> 语境化输出
-> 更新掌握度、retrievalCount、下次复习时间
```

## 验收点

- 领域测试验证练习报告会生成 `memoryTask`。
- API 测试验证 `/api/practice/choice-report` 返回可复习的主动回忆任务。
- E2E 测试覆盖阅读错因进入复习队列，并完成三步复习流程。
